# Релиз-ноты

## 2026-06-11

### Форматирование Telegram-дайджеста

- Заменили отправку обычным текстом в Telegram на HTML-форматирование.
- Добавили жирные заголовки для разделов дайджеста: `Company Updates`, `PM Lens`, `Influencers`, `Industry RU`, `Industry US` и `Watchlist`.
- Добавили поддержку Markdown-заголовков: строки вроде `## Company Updates` теперь отправляются как чистые жирные заголовки Telegram без видимых символов `#`.
- Нормализовали строки со списками в аккуратный формат маркеров, удобный для Telegram.
- Конвертировали простые Markdown-маркеры жирного текста, например `**Acme:**`, в жирный текст Telegram.
- Добавили поддержку цитирования для явно отмеченных строк-цитат через блочные цитаты Telegram.
- Добавили экранирование динамического текста дайджеста перед отправкой как HTML, чтобы символы вроде `<`, `>`, `&` и кавычки не ломали форматирование Telegram.
- Сохранили текущую логику разбиения Telegram-сообщений и форматируем каждый исходный фрагмент уже после разбиения, чтобы не разрезать HTML-теги или HTML-сущности между сообщениями.
- Запрос к Telegram API теперь включает `parse_mode: HTML`.

### Тесты

- Обновили тесты Telegram-отправки: теперь они проверяют HTML-форматированный текст в запросе и `parse_mode: HTML`.
